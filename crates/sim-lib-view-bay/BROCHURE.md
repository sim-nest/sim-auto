# sim-lib-view-bay

In one line: A modeled repair bay screen that turns automotive work into one
portable Scene with matching operator actions.

## What it gives you

It gathers the vehicle, selected site, faults, repair note, parts cart, coding
state, flash gate, and job timeline into one value a SIM surface can render. The
same package also names the operator actions the bay accepts, so a tablet, TUI,
or browser can drive the modeled shop flow through checked Intent values.

## Why you will be glad

Automotive work stops being scattered across separate readouts. The technician
sees the diagnosis, repair path, parts, and guarded effect state together, while
dangerous requests stay visibly gated. Tests can inspect the full bay value
without hardware, accounts, or licensed workshop data.

## Where it fits

It sits above the automotive core, diagnostics, repair information, parts, and
vendor manifests, and below any concrete SIM surface. It is the reusable bay
projection that a command line, TUI, or web session can load.
